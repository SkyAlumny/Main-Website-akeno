Setting up the Mute Role

SkyAlumny

April 25, 2021 • 5 min read

Before you can use the moderative command Mute, you have to set which role is going to be the Mute role. If you haven't tried anything of the sort yet, we'll walk you through it

Decide a Mute Role

Decide which role you are going to use when using Akeno's mute command. The Color RED gives it a dangerous feeling, and the name Muted gives it justice. However, you can just pick any name and color you want. It's all up to you.

Set or Check the Role Permissions.

Go to Server Settings -> Roles -> The Role you're planning to set as Mute Role -> Scroll Down to "Send Messages Permission" -> Disable.




Make sure the mute role is above all other member's role because other role permissions will override it. Just make sure all moderative permissions (ADMINISTRATOR, Kick Members, Ban Members, Manage Messages, etc.) are turned off. See to it also that this role is below Akeno's highest role. For effeciency, place this role under Akeno's highest role.


You can also disallow muted members from connecting to voice channel by disabling 'Connect' from the mute role's permission.


If you are well versed in managing role permissions, you may set any disabling permissions to the muterole, and it will take effect.
If you haven't created a role for this yet, create one and just follow the preceeding procedures.

Have Akeno Recognize the Muterole.

You can now have Akeno recognize this muterole by using the command ak!setmute . Akeno will be using this role to add to users whenever the mute command is called.



You can also check which role the muterole is currently set to by using the same command without parameters
You can now use the moderative command ak!mute in your server!
